## Описание
Консольная утилита для фильтрации содержимого текстовых файлов.
Программа принимает один или несколько входных файлов, содержащих строки, целые числа и вещественные числа (разделённые переводом строки), также:
- записывает целые числа в отдельный файл
- записывает вещественные числа в отдельный файл
- записывает строки в отдельный файл
- выводит статистику по отфильтрованным данным

## Сборка проекта
В корне проекта выполнить:
```mvn clean package```
После успешной сборки в директории target/ будет создан файл: FileFilter-1.0-SNAPSHOT.jar

## Запуск программы
Общий формат: ```java -jar target/FileFilter-1.0-SNAPSHOT.jar [опции] <входные файлы>```
Пример запуска: ```java -jar target/FileFilter-1.0-SNAPSHOT.jar -s -a -p sample- in1.txt in2.txt```

## Поддерживаемые опции
• -o <path> (путь к директории для выходных файлов)
• -p <prefix> (префикс имен выходных файлов)
• -a (режим добавления (append))
• -s (краткая статистика)
• -f (полная статистика)

## Выходные файлы
По умолчанию создаются
- integers.txt
- floats.txt
- strings.txt
Файлы создаются только при наличии соответствующих данных
Если указан префикс: -p result_ -> файлы будут result_integers.txt, result_floats.txt, result_strings.txt
Если указан путь:-o /some/path -> файлы будут созданы в указанной директории

## Статистика
- Краткая (-s)
Выводится только количество элементов каждого типа
- Полная (-f)
Для чисел: количество, минимальное значение, максимальное значение, сумма, среднее значение
Для строк: количество, длина самой короткой строки, длина самой длинной строки

## Запуск тестов
Для запуска unit-тестов: ```mvn test```

## Особенности реализации:
- файлы создаются только при первой записи
- чтение файлов происходит построчно (BufferedReader)
- архитектура разделена по зонам ответственности:
  • Main - точка входа
  • ArgumentsParser - разбор аргументов
  • FileProcessor - основная логика обработки
  • WriteManager - работа с выходными файлами
  • Statistics - сбор статистики
